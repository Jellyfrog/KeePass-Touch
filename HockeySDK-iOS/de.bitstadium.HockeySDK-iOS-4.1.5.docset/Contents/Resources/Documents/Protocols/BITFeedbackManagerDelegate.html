– allowAutomaticFetchingForNewFeedbackForManager:

Can be implemented to control wether the feedback manager should automatically fetch for new messages on app startup or when becoming active.

- (BOOL)allowAutomaticFetchingForNewFeedbackForManager:(BITFeedbackManager *)feedbackManager

Parameters

feedbackManager

The feedbackManager which did detect the new messages

Discussion

By default the SDK fetches on app startup or when the app is becoming active again if there are already messages existing or pending on the device.

You could disable it e.g. depending on available mobile network/WLAN connection or let it fetch less frequently.

– preparedItemsForFeedbackManager:

This optional method can be implemented to provide items to prefill the FeedbackComposeMessage user interface with the given items.

- (nullable NSArray *)preparedItemsForFeedbackManager:(BITFeedbackManager *)feedbackManager

Parameters

feedbackManager

The BITFeedbackManager instance that will handle sending the feedback.

Return Value

An array containing the items to be attached to the feedback message

Discussion

If the user sends the feedback message, these items will be attached to that message.

All NSString-Content in the array will be concatenated and result in the message, while all UIImage and NSData-instances will be turned into attachments.

– forceNewFeedbackThreadForFeedbackManager:

Indicates if a new thread should be created for each new feedback message

- (BOOL)forceNewFeedbackThreadForFeedbackManager:(BITFeedbackManager *)feedbackManager

Return Value

A BOOL indicating if each feedback message should be sent as a new thread.

Discussion

Setting it to YES will force a new thread whenever a new message is sent as opposed to the default resume thread behaviour.
